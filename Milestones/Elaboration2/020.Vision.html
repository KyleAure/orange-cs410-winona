Vision

This is the vision document for the team project. This document introduces the project and describes the core project requirements, key features, and main constraints.

Project introduction

Create a puzzle slider game that allows users to select a photo to be scrambled, only to be put back together by sliding tiles around. When the user exits the game, the puzzle progress is saved for them to continue to finish at a later time. Introduce other game design features to vary game-play options and allow users to save their game for later completion.

Core requirements

This game will need to meet the following core requirements to be considered successful in meeting our customer needs.

  • Store and retrieve player data.
  • Player data authentication and security.
  • User Interfaces
    • Login or set up new player screen.
    • Settings menu.
    • Main game-play screen.
  • Integration with operating system file chooser.
  • Image processing - crop to square, divide, and scramble.
  • Time tracking.

Key features

The following key features are outside the core requirements, but will add value. These features will add value by making our game more competitive against similarly designed games, and offering more customization for our customers.

  • Custom Game-play: The user will be able to use their own pictures.
  • Difficulty: The user will be able to change the games difficulty to their liking. Easy will have less square tiles to slide around, whereas difficult will have more, etc.
  • High Score: The game will keep track of your high score (lowest time to complete), and notify you when you beat your old one.
  • User settings: User will be able to change the theme of the game interface. User will also be able to select a photo from a set of stock photos or upload one of their own.

Main constraints

The game will be created with the following constraints.

  • The software must be written in Java
  • The software must be developed within Rational Software Architect
  • The software must have a GUI interface
  • The artifacts must be sorted in a Bluemix GIT repository
  • The software must store persistent state so that it can be run at a later time
