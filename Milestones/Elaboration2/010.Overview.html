Overview

This section gives an overview of the contents of the Elaboration Phase 2 document

The documentation completed during the Elaboration Phase 2 is to help our team establish the foundations for how our code will be written. To accomplish this goal our team will further explore external entities with which the game will interact (actors), and redefine the nature of this interaction in a high-level vision document. We will be creating diagrams and prototypes that help us better understand how to create a well designed piece of software. A successful Elaboration Phase 2 document will include the following:

  • An updated vision document
  • Additional use-case models
  • Proof of initial code and testing
  • Build and system architecture elements
  • UML diagrams
  • Supplementary requirements

At the end of the Elaboration Phase 2 we should have a better understanding of our project scope, constraints, requirements, and a timeline towards completion.
