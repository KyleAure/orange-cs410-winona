Overview

This section give an overview of the contents of the Transition document

The documentation completed during the Transition phase is to help the team finalize construction of the project and to work towards a final release by finding all remaining issues and bugs. A successful Transition document will include the following:

  • Completed users guide
  • Completed use case models
  • Completed Glossory
  • Completed system architecure
  • Completed UML diagrams
  • Completed application code
  • Test code
  • Contribution summary

At the end of the Transition phase we should have the project completed and ready for release.
